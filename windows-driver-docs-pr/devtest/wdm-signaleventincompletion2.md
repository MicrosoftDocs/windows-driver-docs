# SignalEventInCompletion2 rule (wdm)


The **SignalEventInCompletion2** rule specifies that when processing an asynchronous IRP, the driver needs to call the [**KeSetEvent**](/windows-hardware/drivers/ddi/wdm/nf-wdm-kesetevent) in the completion routine when the **Irp-&gt;PendingReturned** flag is set.

In this case, the completion routine will not be called.

**Driver model: WDM**
